# Plan: Issue #3 — if 内の条件式にフィルタチェーンを適用可能にする

## Goal

`{{#if age | is_neg}}` や `{{#if age | mod(4) | eq(2)}}` のように、if 条件式にフィルタチェーンを適用できるようにする。

## Approach

既存のフィルタインフラ（`string_filter`/`int_filter`/`float_filter` 列挙型、パース関数、適用関数、VM オペコード）を if 条件にも流用する。新しいフィルタ `is_neg` と `eq(n)` を追加する。

## Step 2: `bc_opcode` にフィルタオペコードを追加

**File:** `include/injamm/bytecode.hpp:53-61`

`filter_int_numify` の後に追加:
```cpp
filter_int_is_neg,  // 負数判定
filter_int_eq,      // 等価判定（引数: 比較値）
emit_if_filtered,   // フィルタ適用済み値の if 分岐（filtered_value_ で判定）
```

**設計方針:** フィルタなし `{{#if var}}` は既存の `emit_if`（`for_each_field` による型安全な truthiness 判定）をそのまま使い、フィルタ付き `{{#if var | ...}}` は専用の `emit_if_filtered`（`filtered_value_` の文字列で判定）を使用する。これによりフィルタなしの if に一切のオーバーヘッドが乗らない。

## Step 9: バイトコードコンパイラで if にフィルタ命令を発行

**File:** `include/injamm/bytecode_compile.hpp:218-240`

`compile_if()` を拡張:
1. `expr` を `split_by_pipe()` で分割しフィルタをパース
2. `bc_var_ref` にフィルタを格納
3. フィルタがある場合:
   - `resolve_filtered` → `filter_*` 命令列を発行
   - `emit_if_filtered` 命令を発行（`emit_if` ではなく専用オペコード）
4. フィルタがない場合:
   - 既存の `emit_if` 命令をそのまま発行（変更なし）

## Step 10: バイトコード VM に `is_neg` / `eq` ハンドラを追加

**File:** `include/injamm/bytecode_exec.hpp`

**Computed goto:**
- `L_filter_int_is_neg`: `filtered_value_` を数値化し、負なら `"true"`、そうでなければ `"false"` をセット
- `L_filter_int_eq`: `filtered_value_` を数値化し、`operand` と比較して `"true"`/`"false"` をセット

- ディスパッチテーブルに `&&L_filter_int_is_neg`, `&&L_filter_int_eq`, `&&L_emit_if_filtered` を追加（`&&L_halt` の前）
- `L_halt` のインデックスを 45 → 48 に更新

**Switch-case:**
- `case bc_opcode::filter_int_is_neg:` / `case bc_opcode::filter_int_eq:` / `case bc_opcode::emit_if_filtered:` を追加

## Step 11: `L_emit_if` は変更なし、`L_emit_if_filtered` を新規追加

**File:** `include/injamm/bytecode_exec.hpp`

**既存の `L_emit_if` はそのまま**（フィルタなし if 用、`for_each_field` による型安全な truthiness 判定）。

**新規 `L_emit_if_filtered`**（フィルタ付き if 用）:
```cpp
L_emit_if_filtered: {
  auto const& instr = bc_.instructions[pc];
  bool cond = !filtered_value_.empty() && filtered_value_ != "false" && filtered_value_ != "0";
  if (!cond) {
    pc = instr.operand;
  } else {
    ++pc;
  }
  DISPATCH();
}
```

`case bc_opcode::emit_if_filtered:` も同様のロジックを switch-case 版に追加。

## Step 12: `apply_int_filter` に `is_neg` / `eq` ケースを追加

**File:** `include/injamm/ct_render.hpp:192-335`

`apply_int_filter()` の switch に追加:
```cpp
case int_filter::is_neg: {
  try {
    if (str.find('.') != npos || str.find('e') != npos || str.find('E') != npos) {
      str = std::stod(str) < 0 ? "true" : "false";
    } else {
      str = std::stoll(str) < 0 ? "true" : "false";
    }
  } catch (...) { str = "false"; }
  break;
}
case int_filter::eq: {
  try {
    auto target = entry.arg;
    if (str.find('.') != npos || str.find('e') != npos || str.find('E') != npos) {
      str = (static_cast<long long>(std::stod(str)) == target) ? "true" : "false";
    } else {
      str = (std::stoll(str) == target) ? "true" : "false";
    }
  } catch (...) { str = "false"; }
  break;
}
```
