# Plan: Add numeric comparison filters (eq, ne, gt, gte, lt, lte)

## Overview

Add 6 comparison filters for numeric values to the `int_filter` family. These filter operates on the string representation of a value (parsed via `from_chars`), compare it against a compile-time integer argument, and output `"true"` or `"false"`. Both Bytecode VM (`engine<T>`) and NTTP compile-time (`render<fixed_string>`) are supported, since they share the same `apply_int_filter()` function.

The filter argument is the *right-hand side* of the comparison. For example `{{age | gt(18)}}` outputs `"true"` if `age > 18`.

### 3. `include/injamm/filters.hpp` — apply_int_filter()

Add 5 new `case int_filter::ne:` / `case int_filter::gt:` / etc. blocks. Each follows the identical pattern from `int_filter::eq`:
1. Check if the string contains `.` / `e` / `E` → parse as `double`, compare with `static_cast<double>(entry.arg)`
2. Otherwise → parse as `long long`, compare with `static_cast<long long>(entry.arg)`
3. Set `str = "true"` or `str = "false"`
