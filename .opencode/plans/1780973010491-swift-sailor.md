# Issue #6: break, continue の実装

## 概要

Mustache/inja サブセットテンプレートエンジンに `{{#break}}` と `{{#continue}}` を追加する。
`{{#if}}` と同様の `{{#...}}` プレフィックスを使用し、`{{var}}` との混同を回避する。

## 構文

```mustache
{{#items}}
  {{#if @first}}---{{/if}}
  {{#break}}
  {{#continue}}
{{/items}}
```

## 実装手順

### Step 1: オペコード追加 (`bytecode.hpp`)

`bc_opcode` enum の `halt` の前に `emit_break`, `emit_continue` を追加する。

### Step 2: ループ状態に制御フラグ追加 (`loop_state.hpp`)

`loop_state` に `mutable bool break_flag = false`, `mutable bool continue_flag = false` を追加する。
`mutable` は子 executor が `const loop_state*` 経由でフラグをセットするため。

### Step 3: バイトコード VM の `bc_loop_state` にもフラグ追加 (`bytecode_exec.hpp`)

`bc_loop_state` (line 22-31) に同样的に `mutable bool break_flag = false`, `mutable bool continue_flag = false` を追加する。

### Step 4: VM ハンドラ追加 + ループにフラグチェック (`bytecode_exec.hpp`)

1. `dispatch_table` (line 293-343) の `L_halt` の前に 2 エントリ追加
2. `L_emit_break`: `loop_state_->break_flag = true; return {};`
3. `L_emit_continue`: `loop_state_->continue_flag = true; return {};`
4. `L_emit_section` (line 393) の for ループ内で `child_exec.execute_impl()` 後に:
   ```cpp
   if (!r2) return r2;
   if (ls.break_flag) break;
   ```
5. switch fallback のディスパッチにも同様のハンドラ追加

### Step 5: バイトコードコンパイラ (`bytecode_compile.hpp`)

`compile_body()` (line 446) の `inner.starts_with("#")` 分岐内、`key.starts_with("if")` の前に:
```cpp
if (key == "break") {
  bc_.add_instruction(bc_opcode::emit_break);
  continue;
}
if (key == "continue") {
  bc_.add_instruction(bc_opcode::emit_continue);
  continue;
}
```

`compile_body_with_else()` (line 567) にも同様の処理追加。

### Step 6: CT パス (`ct_chunk.hpp`, `ct_parse.hpp`, `ct_render.hpp`)

1. `ct_chunk.hpp`: `ct_chunk_kind` に `ct_break`, `ct_continue` 追加
2. `ct_parse.hpp`: `ct_parse_into()` の `inner.starts_with("#")` 分岐内で break/continue をパース
3. `ct_render.hpp`: `ct_render_chunks` で break/continue チャンクを処理し、`loop_state` のフラグをセット。セクションループにフラグチェック追加

### Step 7: テスト追加 (`tests/test_injamm.cpp`)

- break: 配列ループで3番目で中断
- continue: 配列ループで2番目をスキップ
- if 内の break/continue
- ネストしたセクションでの動作
- セクション外でのパース時エラー（任意）
